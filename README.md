# ATOM Platform — веб-платформа для языка ATOM

Сайт для экспериментального языка программирования **ATOM** (компилятор на C++
лежит в родительском репозитории): онлайн-редактор кода в стиле VS Code,
документация, новости, форум сообщества, отзывы, каталог пользовательских
проектов, чат поддержки на WebSocket и админ-панель с ролями разработчиков.

Микросервисная архитектура: React-фронтенд, Django REST Framework + Channels
бэкенд, PostgreSQL, Redis (кэш + брокер Celery), Celery worker/beat, Nginx как
единый API Gateway перед фронтендом и бэкендом.

## Технологии

- **Frontend:** React 18 + Vite, React Router, Monaco Editor, axios. Без
  TypeScript и CSS-фреймворков — вся стилизация рукописная.
- **Backend:** Django 5 + DRF, Django Channels (WebSocket), JWT
  (`djangorestframework-simplejwt`), OAuth (Google/GitHub через
  `social-auth-app-django`), Celery + Redis, `drf-spectacular` (Swagger).
- **Инфраструктура:** Docker Compose — `postgres`, `redis`, `backend`
  (ASGI-сервер `daphne`), `celery_worker`, `celery_beat`, `frontend`
  (Vite dev-сервер), `gateway` (nginx).

## Структура

```
website/
├── frontend/        # React (Vite) — редактор кода, страницы сайта
├── backend/         # Django REST + Channels, Celery, приложения по доменам
├── nginx/           # конфиг gateway: /api и /ws → backend, остальное → frontend
└── docker-compose.yml
```

## Что есть на сайте

- **Редактор-терминал** (`/editor`) — многопроектный редактор кода ATOM с
  деревом файлов/папок, автосохранением, подсветкой синтаксиса и
  автодополнением (по реальным встроенным функциям компилятора), просмотром
  токенов/AST (мок, формат совпадает с настоящим компилятором), скачиванием
  проекта архивом и пушем на GitHub. Анонимным пользователям доступен только
  без сохранения — об этом сайт явно предупреждает.
- **Новости** (`/news`) — публикуются разработчиками уровня middle+ через
  блочный редактор (1–7 блоков текст+фото на статью, выбор обложки), список с
  поиском, облаком тегов (мультивыбор) и пагинацией, лента собирается в
  мозаику без пустых ячеек при любом количестве карточек на странице.
- **Документация** (`/docs`) и **«Анатомия ATOM»** (`/learn`) — статичные
  страницы с описанием синтаксиса языка и историей разработки компилятора,
  с реальными вставками кода.
- **Сообщество** (`/forum`) — вопросы и ответы в стиле StackOverflow.
- **Отзывы** (`/reviews`) — оценки и предложения по языку с ответами
  разработчиков.
- **Проекты** (`/repositories`) — витрина пользовательских репозиториев.
- **Проблемы** (`/issues`) — публичный список багов сообщества + приватный
  список «мои баги» в профиле.
- **Чат поддержки** — всплывающий виджет на реальном WebSocket (Django
  Channels): бот на правилах отвечает первым, разработчик может подключиться
  к обращению из очереди в админ-панели.
- **Роли и админ-панель** (`/admin-panel`) — три уровня разработчиков
  (junior/middle/senior) с разными правами (отвечать на отзывы → + публиковать
  новости и вести чат поддержки → + раздавать уровни и регистрировать
  разработчиков), у каждого уникальный `developer_key`. Каталог разработчиков
  с поиском — `/developers`.
- Авторизация по email (JWT) и OAuth-редиректы на Google/GitHub.

## Установка и настройка

1. **Docker и Docker Compose** (Desktop на Windows/Mac, либо `docker` +
   `docker-compose-plugin` на Linux).
2. Скопировать единый файл окружения — все переменные backend и frontend
   находятся в одном файле `website/.env`, у каждой переменной в
   `.env.example` есть комментарий, что она значит и где взять значение:
   ```
   cp .env.example .env
   ```
   Обязательно сгенерировать `DJANGO_SECRET_KEY` (команда есть в комментарии
   рядом с переменной). OAuth-ключи Google/GitHub и `VITE_GITHUB_REPO_URL`
   можно оставить пустыми — соответствующие кнопки на сайте просто не
   появятся либо корректно покажут ошибку, остальной сайт не сломается.

## Запуск

```
docker compose up --build
```

- Сайт целиком (через gateway): http://localhost
- Django admin: http://localhost/admin/
- Swagger-документация API: http://localhost/api/docs/

Первая миграция, суперпользователь и демо-данные (новости/форум/отзывы/баги/
проекты — без них соответствующие страницы покажут только запасные демо-карточки):

```
docker compose exec backend python manage.py migrate
docker compose exec backend python manage.py createsuperuser
docker compose exec backend python manage.py seed_demo_data
```

## Остановка

```
docker compose down          # остановить
docker compose down -v       # остановить и удалить volume (потеряются данные Postgres)
```

## Что пока макет, а не настоящий бэкенд

- **Запуск кода** в редакторе — компилятор ATOM не подключён как сервис
  (см. «Core API» ниже), кнопка «Запустить» только показывает заглушку.
- **Токены/AST** в редакторе — наивная мок-генерация (формат совпадает с
  настоящим компилятором, но это не реальный лексер/парсер).
- **ИИ в чате поддержки** — бот отвечает по простым правилам (keyword
  matching), без внешнего LLM-провайдера.

### Core API — главный нерешённый архитектурный вопрос

Как обернуть C++ компилятор ATOM в сервис, который дёргает Django/Celery, —
требует отдельного решения: gRPC-сервис, REST-обёртка (FastAPI/Flask) рядом
с Django, либо Celery-задача, запускающая компилятор в изолированном
окружении (Docker-in-Docker/gVisor/firecracker) и стримящая результат через
WebSocket.
